## Conventions

- All tables use `UUID` primary keys generated by the database (`gen_random_uuid()`).
- All timestamps are `TIMESTAMPTZ` (UTC with timezone offset).
- `tenant_id` appears on every tenant-scoped table as a non-nullable FK to `tenants.id`.
- JSONB columns store structured payloads where the schema evolves independently of the table DDL.
- No hard deletes on audit-related tables. Soft-delete uses `deleted_at TIMESTAMPTZ NULL`.
- Append-only tables (e.g., `category_result_versions`, `reviewer_actions`) have no `UPDATE` permissions for `sf_app`.
