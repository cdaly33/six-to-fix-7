## Objective
Implement admin-only BuildDeployStamp component displaying current deployment version, build timestamp, and environment tag in sidebar; add relative-time formatting utility for human-readable "deployed X minutes ago" strings.

## Changes Summary

### New Components
- `BuildDeployStamp.razor` — Admin-only sidebar component:
  - Displays ApplicationVersion (e.g., "v2.1.0-rc1")
  - Shows DeployedAt timestamp with relative-time formatting (e.g., "deployed 23 minutes ago")
  - Renders environment tag (prod/staging/dev) with visual indicator
  - Hides for non-admin users ([Authorize(Roles = "SuperAdmin")])
  - Queries `/api/deployment-info` on mount

### Formatting Utilities
- `RelativeTimeFormatter.cs` — Static utility class:
  - `Format(DateTime utc, bool includeSeconds = false)` → "X minutes ago" / "X hours ago" / "X days ago"
  - Handles edge cases: "now" for <1 minute, "1 year ago" for >365 days
  - Timezone-aware: converts UTC to user's local time
  - Testable: pure function, no external dependencies

## Standing Technical Decision

**Relative-Time Formatting:** Implemented client-side (C# utility) rather than server-side, because:
1. Reduces per-request server load (no timestamp formatting on every page render)
2. Enables live "ago" countdown if future Blazor enhancements request auto-refresh every 60s
3. Pure function, highly testable, no side effects
